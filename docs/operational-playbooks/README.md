# Operational playbooks (V8.6 — Knowledge Layer)

Short, plain-English how-to guides for running the shop. These mirror the in-app
versions at **/admin/playbooks** (which the morning briefing links to automatically
when an issue arises). Keep this folder for printing and for the owner's binder.

The canonical content lives in `src/lib/shop-intelligence/playbook-content.ts` so the
app and these docs stay aligned — edit there and reflect the change here.

| Playbook | Use it when |
|---|---|
| [Butcher words explained](butcher-words.md) | **New to butchery — start here** |
| [Receiving stock](receiving-stock.md) | Meat arrives from a supplier |
| [Carcass intake](carcass-intake.md) | Breaking an animal into cuts |
| [Recording waste](recording-waste.md) | Something gets thrown away |
| [Correcting stock](correcting-inventory.md) | The books don't match the fridge |
| [Managing orders](managing-orders.md) | Preparing and handing over orders |
| [Supplier compliance](supplier-compliance.md) | Keeping halal certificates current |
| [Handling low stock](handling-low-stock.md) | Deciding what to re-order |
| [Reading your briefing](reading-your-briefing.md) | Using the morning briefing |

> Golden rule (V8.13): the system only ever *suggests*. It never changes stock,
> prices, orders or costs by itself. You decide.
