## 2. Mission

Remove navigation. The operator should not read an action, remember it, leave TODAY,
find the correct page, then search for the item. One tap takes the operator straight to
where the work is done, with the exact item pre-focused and the reason shown.

V15.2 answers the third question every action must answer — **Where do I do it?** — and
removes it from the operator's head. One tap to the *work*, never one tap to *execute*.

---

## 3. What changed

### The contract (new pure layer)
- **`src/lib/owner-brain/action-target.ts`** — the Action Target Contract. Pure, no I/O,
  never mutates the decision. `resolveActionTarget(decision)` reads the same structured
  signals as V15 ranking (`area`, `id`, the recommended-action verb) and returns
  `{ actionType, destination, entityLabel, entitySlug, reason, href }`. Re-exported from
  `owner-brain/brain.ts`.

| Action | Verb | Destination | Item pre-focused |
|---|---|---|---|
| Count *X* today | count | `/admin/stock-count` | yes — slug from id, scroll + ring-highlight |
| Order *X* tomorrow / order less | order | `/admin/purchasing` | yes — matching recommendation, `#anchor` + ring |
| Sell *X* first | sell | `/admin/inventory` | yes — matching batch, scroll + ring-highlight |
| Check *X* / record waste (expired) | fix | `/admin/stock-count` | yes — the single correction door |
| Compliance certificate expired | fix | `/admin/compliance` | banner (compliance is supplier-keyed) |
| Other findings (margin / yield / generic owner-actions) | review | `/admin/today/[id]` | n/a — keeps the existing read page |

### The surfaces
- **TODAY** (`src/app/admin/today/page.tsx`) — Do-now cards and Later rows now link to
  `resolveActionTarget(decision).href` instead of the read-only detail page. The card still
  answers What + Why; V15.2 only fixes Where.
- **Action-context banner** (`src/components/owner-brain/action-context.tsx`) — a shared
  server component rendered atop every destination. Rebuilt entirely from the URL
  (`from`, `do`, `focus`, `why`), so it **survives a refresh** with no server round-trip.
  Names the exact item ("Count Chicken Breast"), shows the reason, and offers an explicit
  **Back to Today** return.
- **Destinations focus the entity (full depth):** `stock-count` and `inventory` take a
  `focusSlug` and auto-scroll + ring-highlight the matching item;
  `purchasing` rings the matching recommendation and links via `#slug` anchor;
  `compliance` shows the banner. All read `searchParams` via the new `firstParam` util.

### Safety (Mission 9)
- Every destination is a read/work screen the operator drives by hand. **No** pre-filled
  values, **no** auto-submit, **no** path that changes stock, prices, orders, discounts or
  compliance on its own. One tap to the work, not to execute it.

No brain/logic changes: `buildOwnerBrain`, the compression engine, confidence routing and
V14 truth are untouched. No new migrations.

### One-tap operator journey (live app)

- **Do-now actions link straight to the work:** carry `from=today` + item focus, never the
  read-only detail page (for actions with a work screen).
- **No action opens the wrong destination:** every one-tap link lands on a known work route
  (`/admin/stock-count`, `/admin/purchasing`, `/admin/inventory`, `/admin/compliance`).
- **Destination knows why the operator arrived:** the "From Today" banner renders, naming
  the action/item.
- **Context survives a refresh:** banner still present after `reload()` (it lives in the URL).
- **Completion returns to TODAY:** Back-to-Today lands on `/admin/today`.
