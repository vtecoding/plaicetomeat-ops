## 2. Mission

Make the intelligence boundary **structural**. The decision engine may grow arbitrarily
sophisticated (scores, confidence, ranking evidence); the operator layer must stay simple
forever. The goal is not to hide fields carefully — it is to make leakage **impossible by
design**. Boundary-hardening only: no ranking/compression/UI behaviour changes, no migrations.

---

## 3. What changed

### Two hard types + one conversion boundary (`types.ts`, new `operator-action.ts`)
- **`ScoredAction`** (internal) = the formal name for the existing scored decision
  (`priority`, `category`, money, `sourceEvidence.basis.confidence`, metrics). Scoring and
  compression operate on it.
- **`OperatorAction`** (external) = the only shape the operator layer receives. It is
  *structurally incapable* of carrying a calculation — every field is a safe display string
  (`id, actionType, title, whatHappened, whyItMatters, recommendedAction, reason, impactLabel,
  impactTone, owner, dueLabel, destination, entityLabel, href, basisSummary, supportingFacts,
  playbook, completion`). No `priority`/`confidence`/`score`/`severity`/`rank`/evidence exists on it.
- **`toOperatorAction(scored)`** — the single allowed path from engine to UI. It reuses
  `resolveActionTarget` (V15.2), maps money → `impactLabel`/`impactTone`, due window → `dueLabel`,
  and **drops `basis.confidence`** (only the plain `basisSummary` survives).
- `ActionType` moved to `types.ts` (re-exported from `action-target.ts`) to avoid a circular import.

### Engine returns an operator-safe brain (`brain.ts`)
- `runDecisionEngine(intel)` (internal) runs the scored pipeline once; `buildOwnerBrain`
  converts at the boundary so `OwnerBrain.doNow`/`later`/`walkSteps` are **`OperatorAction[]`**,
  plus `opportunityCount`. **Removed** from the operator brain: `actionEvidence`, `urgent`,
  `important`, `opportunities` (scored).
- `getDecisionDiagnostics(intel)` (new, internal) exposes the scored picture + ranking evidence
  for audit/dev — never imported by a UI surface (Mission 7).
- `findDecision` returns `OperatorAction`.

### Operator surfaces consume only OperatorAction (Missions 1, 2, 5, 6)
`day.ts` (`buildDayShape(OperatorAction[])`), `briefing.ts` (compliance detected via
`actionType==='fix' && destination==='/admin/compliance'`, not `area`), `today/page.tsx`
(cards/rows/zone/briefing panel; uses `op.href` directly — dropped `resolveActionTarget`),
`decision-detail.tsx`, `guided-day.tsx`, `today/[id]` + `walk` pages. No visual change.
