## 2. Mission

The hardest part of V15 was never building the Owner Brain — it is preventing future
releases from quietly undoing it. V15.5 turns the doctrine from a convention into an
**enforced invariant**: dashboards, metrics, scores, ranking values and operator thinking
cannot creep back in without failing the build. This is a **protect-the-architecture**
release — no new intelligence, no new features (see Non-Goals).

---

## 3. What changed

### New continuous-reduction guard (`scripts/verify-owner-brain-compliance.mjs`)
A static guard (no app/DB) covering four spec missions, run as `verify:owner-brain-compliance`:

- **Mission 2 + 8 — Dashboard Regression Guard.** Scans operator surfaces (comments
  stripped) for dashboard language: percentages, `KPI`, charts, and confidence / priority /
  score / ranking **values** (`Confidence 82%`, `Priority 93`, `Health Score`, `ranked #1`).
  The strict decision/presentation layer (TODAY, Later, Morning Briefing, decision-detail,
  guided-day, action-context) additionally forbids any bare `%` and the word `variance`. The
  one-tap work surfaces (stock-count, purchasing, inventory, compliance, counter) are allowed
  their honest figures (e.g. `+1.2 kg vs system`) but still no scores. Also fails on any new
  metric/chart `data-testid` (a new dashboard panel sneaking in).
- **Mission 3 — Three-Action Rule.** Asserts `DO_NOW_MAX = 3` is permanent, that the Do-now
  slice is capped by it (`ranked.slice(0, DO_NOW_MAX)`), and that nothing anywhere raises it.
- **Mission 7 — Action Pipeline Seal.** Verifies the pipeline stages are wired with no
  bypass: Candidates (`toOwnerDecision`) → Scoring (`rankDecisions`) → Competition/Compression
  (`compressActions`) → Execution/Presentation (`toOperatorActions(engine.doNow)`), and that
  the operator's Do-now is fed **only** by the compression boundary.

The patterns are proven non-vacuous: they catch all six forbidden examples from the spec
(`Revenue +12%`, `Confidence 82%`, `Priority 93`, `Variance`, `Trend Score`, `Health Score`)
and clear all allowed strings, including the look-alike identifiers `dateConfidence`,
`CounterDashboard` and `stockVarianceKg`.

### Living doctrine report (`docs/v15/Owner-Brain-Compliance.md`)
Regenerated on every guard run (so it can never go stale). Contains the automated results
plus the structured-review missions that cannot be automated:

- **Mission 1 — Doctrine surface audit:** every operator surface confirmed to require no
  interpretation, calculation, training or prioritisation.
- **Mission 4 — Ten-second rule:** an experienced butcher understands what matters in ≤10s
  (≤3 numbered Do-now cards above the fold, single plain verbs).
- **Mission 5 — One-minute rule:** a new operator understands the day in ≤1 min from the
  Morning Briefing + Do Now alone, no training.
- **Mission 6 — Cognitive load audit:** navigation / interpretation / prioritisation /
  configuration / search decisions on TODAY = 0; the operator makes at most three decisions.
- Risk areas (legacy "Dashboard" headings, the allowed `/admin` Business Insights hub, new
  surfaces) and future recommendations.

### Gate wired (`package.json`)
Added `verify:owner-brain-compliance` to the script set so it runs alongside the existing
verify gates.

**No operator-facing code, no engine behaviour, and no migrations changed.** TODAY is byte
-for-byte the same as V15.4.
