## 1. Executive summary

V12.2 makes every staff/admin action authenticated, authorised, branch-scoped,
session-bound, and tamper-resistant, and removes the silent production
fallbacks that previously let authority degrade quietly.

What changed, at a glance:

- **One authority path.** `requireStaffContext` / `resolveStaffContext` is now the
  single staff/admin gate. Every admin page, the counter, the shared ops-capture
  shell, and all eight privileged server actions resolve authority through it. The
  scattered `requireManager`/`requireOwner`/`requireBranchStaff` copies now delegate
  to it.
- **No branch fallbacks.** Every `profile.branchId ?? getPublicBranch().id`,
  `?? demoBranch.id`, and owner `?? ""` is gone. Branch-scoped surfaces require a
  real, non-null branch and fail closed (→ `/unauthorised`) otherwise — for owners too.
- **Explicit owner authority.** `/admin/audit` and `/admin/releases` re-check
  `owner` in the page/data path; the releases actions require owner via the shared
  resolver. Managers can no longer reach them even if middleware is bypassed.
- **Signed session envelope.** The staff activity cookie is now an HMAC-SHA256,
  timestamped, user-bound envelope. A **missing** cookie is treated as **expired**
  (was fail-open); idle **and** absolute timeouts are enforced; tampered or
  cross-user envelopes are rejected as invalid.
- **Hardened logout.** `signOut()` failures are surfaced to the user, never swallowed.
- **Hardened login.** Lockout now spans **account + network** (looser network
  budget for shared NAT) using the existing rate-limit primitives; the network
  signal is stored as a salted hash, never a raw IP.
- **No PII in auth logs.** Email and profile rows removed from login failure logs.
- **Dead route removed.** The unused `/compliance` matcher/route-authority entry is gone.

Gate: **typecheck ✓, lint ✓ (2 pre-existing-style warnings), 327 unit tests ✓,
build ✓**, plus three live-DB adversarial regression suites green.

## 3. Authority model changes

- **Single path.** `resolveStaffContext(minRole, { branchScoped })` returns
  `{ ok:true, profile, branchId }` or a typed refusal (`unauthenticated` |
  `forbidden` | `no_branch`). `requireStaffContext` wraps it for pages and
  redirects (`unauthenticated → /login`, else `→ /unauthorised`).
- **Role hierarchy.** `ROLE_RANK` + `hasMinRole` make "at least manager" / "owner
  only" explicit; a missing role always fails closed; owner outranks all, so
  owner-global access is deliberate, not accidental.
- **Owner revalidation.** `/admin/audit` and `/admin/releases` call
  `requireStaffContext("owner")` in the server page (the data path), and the
  releases actions require owner — independent of middleware.
- **Branch isolation.** `isBranchAuthorised(role, profileBranchId, target)` —
  owner branch-global; everyone else only their own non-null branch — backs the
  counter branch guard, so branch A cannot reach branch B.

## 4. Session model changes

- **Envelope** `{ uid, iat, seen }`, HMAC-SHA256 signed, stored in the existing
  `ptm_staff_last_seen` HttpOnly cookie.
- **Missing = expired.** Absence is no longer silently accepted.
- **Idle timeout** 4h (sliding `seen`); **absolute timeout** 12h (fixed `iat`).
  Both enforced every request.
- **Binding.** `uid` must match the Supabase `getUser()` id; a cross-user envelope
  is rejected as invalid *before* any timeout verdict.
- **Tamper-resistance.** Constant-time signature check; tampered payload / bad
  signature / malformed token → invalid.
- **Edge-safe.** Web Crypto only (no `node:crypto`), so it runs in the Edge
  middleware. Secret: `STAFF_SESSION_SECRET` → `ORDER_ACCESS_SECRET` → dev
  fallback; production hard-fails if unset/short.

## 5. Branch-isolation changes

- All ~17 production branch fallbacks removed (admin pages, `/admin/orders`,
  `/counter`, ops-capture shell, owner `""`). `getPublicBranch()` remains only for
  the **public storefront** and for cosmetic branch display on Settings; the
  authoritative branch id always comes from the signed-in profile.
- Branch-scoped surfaces fail closed on a null branch for **every** role,
  including owner (an owner with no branch is sent to `/unauthorised` rather than
  silently bound to the first active branch).

## 7. Remaining risks

- **Live valid-cookie theft** is still possible (inherent to cookie auth); now
  bounded by idle (4h) + absolute (12h) timeouts and user-binding, not eliminated.
- **Production profiles must carry `branch_id`.** With fallbacks removed, any
  manager/owner whose `branch_id` is null will be refused branch-scoped screens.
  Operator step: confirm prod profiles have a branch before deploy. (Failure mode
  is safe — refusal, not data leak.)
- **Absolute timeout (12h)** forces a daily-ish re-login; chosen for a single-shop
  day. Tune via `STAFF_SESSION_ABSOLUTE_TIMEOUT_MS` if it annoys the owner.
- **Network lockout behind shared NAT**: budget set loose (20/window) to avoid
  false positives; revisit if real abuse appears.
- **Secret rotation**: rotating `STAFF_SESSION_SECRET`/`ORDER_ACCESS_SECRET`
  invalidates live envelopes (forces re-login) — acceptable, but note for ops.
- **`admin-compliance-client.tsx` client `branchId ?? ""`**: a client-side form
  default for an already-branch-scoped supplier record; the server RPC remains the
  branch gate. Low risk; left as-is (out of server-authority scope).
- RLS/SQL authority unchanged in V12.2; the app-layer authority sits on top of the
  existing database backstop.
