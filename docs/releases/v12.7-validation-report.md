# V12.7 Validation Report - Operational Truth Layer

Date: 2026-06-07
Scope: V12.7 only. No V12.8/V12.9 implementation, no observability slice, no pricing provenance slice, and no production certification.
Baseline: `docs/v12.7-discovery-report.md`.
State: implemented and validated locally.

## 1. Executive Summary

V12.7 replaces silent production demo fallbacks with explicit operational truth states across the highest-risk owner and storefront surfaces. The app now has a typed `DataResult` contract, runtime truth helpers for production/demo behavior, a shared `OperationalSnapshotV1` for `/admin` and `/admin/today`, and page-level truth banners for degraded, unavailable, no-data, and configuration-required states.

Closed gaps:
- production storefront branch resolution now requires an explicit canonical branch id
- catalog, pickup-window, counter-order, operations-intelligence, and release fallback behavior is gated by explicit demo allowance
- public storefront pages render honest unavailable/configuration states instead of silently presenting demo data
- `/admin` and `/admin/today` consume the same operational snapshot truth state
- dashboard zero-order state is represented as `NO_DATA`
- partial intelligence loader errors are elevated to `DEGRADED`
- missing production configuration is elevated to `CONFIGURATION_REQUIRED`

## 3. Truth State Contract

`src/lib/domain/data-result.ts` adds explicit states:
- `HEALTHY`
- `NO_DATA`
- `DEGRADED`
- `UNAVAILABLE`
- `UNAUTHORISED`
- `CONFIGURATION_REQUIRED`

The contract is intentionally small and plain. Result-returning loaders can carry both successful data and the operational truth needed to render honest UI messages.

## 4. Runtime / Demo Fallback Policy

`src/lib/server/runtime-truth.ts` centralises:
- production detection
- explicit demo fallback allowance
- canonical storefront branch configuration

Production storefront branch resolution requires `CANONICAL_BRANCH_ID` or `NEXT_PUBLIC_CANONICAL_BRANCH_ID`. Development/test can still use demo-friendly behavior, but that behavior is routed through `allowDemoFallback()` instead of being implicit in production loaders.

## 5. Loader Changes

Implemented result-returning loaders:
- `getPublicBranchResult`
- `getBranchSettingsResult`
- `getActiveCategoriesResult`
- `getAllCategoriesResult`
- `getPublicProductsResult`
- `getPublicProductBySlugResult`
- `getAllProductsResult`
- `getPickupWindowsResult`
- `getCounterOrdersResult`

Compatibility wrappers remain for existing call sites, but the production-facing storefront and admin surfaces now use the result variants where truth state matters.

Demo fallback is now explicitly gated in:
- catalog loaders
- pickup-window loaders
- counter-order loaders
- operations intelligence
- releases governance

## 6. UI Truth Surfaces

Updated owner/admin surfaces:
- `/admin` consumes `getOperationalSnapshotV1`
- `/admin/today` consumes `getOperationalSnapshotV1`
- both render `data-testid="truth-state-banner"` when state is not healthy

Updated public storefront surfaces:
- `/`
- `/shop`
- `/product/[slug]`
- `/checkout`
- `/our-halal-promise`
- `site-header`

These pages render `data-testid="public-truth-state"` for unavailable/configuration-required branch or catalog states instead of silently substituting demo data.

The countdown banner no longer imports demo pickup windows. It now presents a static, honest message that checkout shows live pickup windows from shop settings.

## 9. Remaining Risks

- Some lower-risk loaders still return empty arrays on failure outside the V12.7 owner/storefront scope. They should be handled in a future data-truth hardening pass.
- Compatibility wrappers still return default-shaped data for legacy call sites. The high-risk production surfaces were moved to result-returning variants, but remaining callers should be migrated gradually.
- Local checkout HTTP verification was skipped because no running `CHECKOUT_BASE_URL` was configured. DB/RPC checkout integrity checks passed.
- This slice does not certify production readiness. V12.8/V12.9 and the final certification gate remain out of scope.
